================================================================================
❓ PREGUNTAS FRECUENTES
================================================================================

P: ¿Debo usar ECS o EC2?
R: ECS es mejor (serverless), pero EC2 es más simple si es tu primera vez.

P: ¿Dónde guardo las contraseñas/secrets?
R: USA AWS Secrets Manager o Systems Manager Parameter Store.
   NUNCA los guardes en .env en producción.

P: ¿Qué pasa si un test falla?
R: Revisa los logs, ajusta la configuración y vuelve a ejecutar pytest.

P: ¿Necesito cambiar la SECRET_KEY después del primer deploy?
R: NO. Si la cambias, invalidarás todas las sesiones de usuario.

P: ¿Cómo actualizo mi aplicación en producción?
R: 1. git pull
   2. docker-compose build
   3. docker-compose down && docker-compose up -d
   O si usas ECR: actualiza la task definition en ECS

P: ¿Dónde veo los logs en producción?
R: En AWS CloudWatch. Sigue la guía para configurar.
